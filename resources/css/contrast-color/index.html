The .card above should have readable (white) text inside, as its color is set using the color-contrast() function, passing in the background color. It's guarded by a @supports() rule to make sure the browser can do this.

If there's no readable text, the browser seems to support the function, but apparently decides to just return the color that was passed in.

If the text is yellow, the browser either doesn't understand @supports(), or it correctly understands it, but doesn't support the contrast-color() function.
